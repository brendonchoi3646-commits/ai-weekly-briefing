# AI 주간 브리핑 생성 스킬

와이어로프 회사 기획팀이 매주 전 직원에게 배포하는 AI 트렌드 브리핑 HTML을 자동으로 생성한다.

---

## 저장 경로 및 파일명

- **저장 폴더**: `C:\Users\기획팀\AI뉴스브리핑\`
- **파일명**: `AI브리핑_YYYY-MM-DD.html` (오늘 날짜 기준)
- **GitHub 리포**: `brendonchoi3646-commits/ai-weekly-briefing`

---

## 1단계: 뉴스 검색 (4개 쿼리 동시 실행)

아래 4개 검색을 **반드시 동시에** 실행해 시간을 줄인다.
검색은 모두 **국내 뉴스 중심**, **최근 1주일** 기준으로 한다.

| 섹션 | 검색 키워드 예시 |
|------|----------------|
| 핵심 AI 뉴스 | `YYYY년 M월 AI 뉴스 ChatGPT Claude Gemini 한국` |
| 국내 기업 동향 | `YYYY년 M월 삼성 LG 현대 SK AI 도입 사례` |
| 제조·와이어로프 AI | `YYYY년 제조업 AI 철강 와이어로프 품질검사 스마트팩토리` |
| 매크로 트렌드 | `YYYY년 M월 AI 정책 투자 규제 트렌드 한국` |

신뢰도 높은 출처를 우선한다: `zdnet.co.kr`, `etnews.com`, `aitimes.com`, `bloter.net`, `steeldaily.co.kr`, `bizinfo.go.kr`, `mss.go.kr`, `korea.kr`

개인 블로그나 출처 불명 사이트(예: 개인 도메인, 비공식 블로그)는 피한다.

---

## 2단계: HTML 생성

검색 결과를 바탕으로 아래 **섹션 구성**에 맞게 HTML을 작성한다.

### 섹션 구성

| # | 섹션 | 아이콘 | 뉴스 항목 수 |
|---|------|--------|------------|
| 1 | 이번 주 AI 핵심 뉴스 | 🔵 | 3건 |
| 2 | 국내 기업 AI 도입 동향 | 🟢 | 3건 |
| 3 | 제조·와이어로프 분야 AI | 🟠 | 3건 |
| 4 | AI 매크로 트렌드 | 🟣 | 2건 |

각 항목은 **배지 + 제목 + 2~3줄 요약 + 출처(링크·날짜)** 형식으로 작성한다.
각 섹션 하단에는 **시사점 박스**를 추가해 실무적 의미를 한 줄로 요약한다.

### 배지 종류

- `NEW` (빨강): 이번 주 새로 나온 소식
- `HOT` (주황): 주목도 높은 이슈
- `동향` (파랑): 흐름·트렌드성 기사
- `정책` (초록): 정부·규제 관련

### HTML 디자인 원칙

- 뉴스레터 스타일, 모바일 반응형
- 헤더: 다크 네이비 그라데이션 배경, 발행일·예상 읽기 시간 표시
- 섹션별 카드 형태, 섹션 색상 구분 (파랑/초록/주황/보라)
- 폰트: `'Apple SD Gothic Neo', 'Noto Sans KR', 'Malgun Gothic', sans-serif`
- 최대 너비 720px, 본문 배경 `#f4f5f7`
- 예상 읽기 시간: 약 7분

### HTML 구조 요약

```
헤더 (회사명, 날짜, 읽기 시간)
└── 섹션 탐색 바 (4개 섹션 태그)
    └── 섹션 1~4 (카드)
        └── 섹션 헤더 (아이콘, 제목, 건수)
            └── 뉴스 항목들 (배지, 제목, 요약, 출처)
            └── 시사점 박스
푸터 (기획팀 제공, 발행일)
```

## 3단계: 파일 저장

생성한 HTML을 아래 경로에 저장한다.

```
C:\Users\기획팀\AI뉴스브리핑\AI브리핑_YYYY-MM-DD.html
```

날짜는 오늘 날짜(브리핑 작성 기준일)를 사용한다.

---

## 4단계: GitHub 커밋 (선택)

사용자가 명시적으로 요청한 경우 커밋·push한다. 기본은 파일 저장까지만.

```bash
cd "C:\Users\기획팀\AI뉴스브리핑"
git add "AI브리핑_YYYY-MM-DD.html"
git commit -m "feat: AI 주간 브리핑 YYYY-MM-DD 호 발행"
git push
```

## 품질 체크리스트

HTML 생성 후 아래 항목을 자체 검수한다:

- [ ] 모든 출처 링크가 실제 URL인가 (추측 URL 금지)
- [ ] 출처마다 날짜가 표기되어 있는가
- [ ] 섹션 3 항목이 3건인가
- [ ] 기사 날짜와 본문 시제가 일치하는가 (미래 기사를 과거형으로 쓰지 않았는가)
- [ ] 신뢰도 낮은 출처(개인 블로그 등)가 없는가
